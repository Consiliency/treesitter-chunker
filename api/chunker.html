chunker.chunker.chunk_file_with_token_limit(path, language, max_tokens, model='gpt-4', extract_metadata=True)[source]

Parse file and return chunks that respect token limits.

This function chunks a file using tree-sitter and ensures no chunk exceeds the specified token limit. Large chunks are automatically split while preserving code structure when possible.

Parameters:

  • path (str | Path) – Path to the file to chunk

  • language (str) – Programming language

  • max_tokens (int) – Maximum tokens per chunk

  • model (str) – Tokenizer model to use (default: “gpt-4”)

  • extract_metadata (bool) – Whether to extract metadata (default: True)

Return type:

list[CodeChunk]

Returns:

List of CodeChunk objects with token counts in metadata

Examples

Basic usage:

from chunker.chunker import chunk_file

# Chunk a Python file
chunks = chunk_file("example.py", "python")

for chunk in chunks:
    print(f"{chunk.node_type}: {chunk.start_line}-{chunk.end_line}")

Parallel processing:

from chunker.chunker import chunk_files_parallel

files = ["file1.py", "file2.py", "file3.py"]
results = chunk_files_parallel(files, "python", max_workers=4)

for file_path, chunks in results.items():
    print(f"{file_path}: {len(chunks)} chunks")
